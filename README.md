# -LeetCode-100-
リートコード（LeetCode）で最も人気のある100問の学習記録：コードの書き方を学び、同時にアルゴリズムに関する内容を復習する
# リートコードで問題を解く心得  

1. **第一回目：コードを追いながら入力する**  
   - 他人の解法を参考にし、**アルゴリズムの流れ（思路）を理解する**。  
   - コードの構造や変数名の意味を深く考える。  

2. **第二回目：完全自力で再実装する**  
   - **ヒントや参考コードを一切見ずに**、自分の理解で書き直す。  
   - エラーが出たら「**どこが理解不足か？**」を重点的に振り返る。  

3. **エラーは成長のチャンス**  
   - コンパイルエラーやロジックミスは、**自分の思考の盲点**を明らかにする。  
   - 修正する過程で、**より深い理解**に到達できる。  

> **名言**：  
> "Error is not a failure, it's a feedback."  
> （エラーは失敗ではなく、フィードバックである）
